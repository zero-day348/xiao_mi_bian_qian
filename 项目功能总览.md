# 小米便签项目功能总览（按功能链路）

本文档从“用户操作”出发，串联本项目的核心业务链路：新建、编辑、提醒、搜索、同步、挂件。

---

## 1. 新建便签链路

### 1.1 用户入口
- 列表页点击“新建便签”按钮：`NotesListActivity#createNewNote()`
- 编辑页菜单“新建便签”：`NoteEditActivity#createNewNote()`
- 桌面挂件未绑定内容时点击挂件：`NoteWidgetProvider#update(...)` 中走 `Intent.ACTION_INSERT_OR_EDIT`

### 1.2 页面与对象初始化
- 编辑页初始化入口：`NoteEditActivity#initActivityState(Intent)`
- 新建场景构造工作对象：`WorkingNote#createEmptyNote(...)`
- 内部会初始化：
  - 目标目录 `folderId`
  - 挂件信息 `widgetId/widgetType`（若有）
  - 背景色 `bgColorId`

### 1.3 首次落库
- 真正创建数据库记录发生在保存时：
  - `WorkingNote#saveNote()`
  - `Note#getNewNoteId(...)`：向 `note` 表插入一条新记录
- 关键字段：
  - `type = TYPE_NOTE`（普通便签）
  - `local_modified = 1`（标记待同步）
  - `parent_id = folderId`（决定所属目录）

### 1.4 用户可见结果
- 列表立即出现新便签（由 `ContentProvider notifyChange` 触发刷新）
- 若从挂件创建，后续会同步更新挂件展示内容

---

## 2. 编辑便签链路

### 2.1 用户入口
- 列表点击某条便签：`NotesListActivity#openNode(...)`
- 搜索结果点击进入：`NoteEditActivity` 通过 `SearchManager.EXTRA_DATA_KEY` 解析目标 noteId

### 2.2 数据加载
- `WorkingNote#loadNote()`：加载 `note` 主表字段（背景、提醒、挂件、修改时间等）
- `WorkingNote#loadNoteData()`：加载 `data` 子表（正文、清单模式、通话字段等）

### 2.3 编辑模式
- 普通文本模式：直接编辑 `EditText`
- 清单模式：`NoteEditActivity#switchToListMode(...)`
  - 每一行由 `NoteEditText + CheckBox` 组成
  - 序列化规则：
    - 已勾选：`√ 文本`
    - 未勾选：`□ 文本`

### 2.4 保存策略
- 离开页面自动保存：`NoteEditActivity#onPause() -> saveNote()`
- 增量写入：
  - `Note` 维护主表增量：`mNoteDiffValues`
  - `NoteData` 维护子表增量：`mTextDataValues / mCallDataValues`
- 最终写库：`Note#syncNote(...)`
  - 先更新 `note`，再更新/插入 `data`
  - 统一更新 `local_modified=1` 与 `modified_date`

### 2.5 用户可见结果
- 列表摘要、修改时间、颜色、清单样式即时生效
- 若便签绑定桌面挂件，编辑后挂件会刷新

---

## 3. 提醒（闹钟）链路

### 3.1 设置提醒
- 编辑页菜单“提醒”：`NoteEditActivity#setReminder()`
- 选择时间对话框：`DateTimePickerDialog` + `DateTimePicker`
- 回调写入：`WorkingNote#setAlertDate(...)`

### 3.2 注册系统闹钟
- `NoteEditActivity#onClockAlertChanged(...)`
  - 构造 `PendingIntent(AlarmReceiver)`
  - `AlarmManager.set(RTC_WAKEUP, date, pendingIntent)`
- 取消提醒则调用 `alarmManager.cancel(...)`

### 3.3 触发提醒
- 广播接收：`AlarmReceiver#onReceive(...)`
- 拉起提醒页：`AlarmAlertActivity`
  - 展示便签摘要
  - 播放系统闹铃
  - 可直接跳转编辑页处理内容

### 3.4 开机恢复
- `AlarmInitReceiver#onReceive(...)`
  - 查询所有未来提醒
  - 逐条重新注册闹钟（防重启丢失）

### 3.5 用户可见结果
- 到点弹窗 + 铃声提醒
- 可直接进入目标便签

---

## 4. 搜索链路

### 4.1 Provider 搜索入口
- `NotesProvider#query(...)` 处理：
  - `URI_SEARCH`
  - `URI_SEARCH_SUGGEST`

### 4.2 查询逻辑
- 使用 `NOTES_SNIPPET_SEARCH_QUERY`
- 条件：
  - `snippet LIKE %keyword%`
  - 排除回收站
  - 仅 `TYPE_NOTE`

### 4.3 结果展示与跳转
- 搜索建议返回 `intent_data` 指向 `TextNote.CONTENT_TYPE`
- 点击结果进入 `NoteEditActivity`
- 编辑页内高亮关键字：`getHighlightQueryResult(...)`

### 4.4 用户可见结果
- 搜索框输入关键字可看到便签候选
- 点击候选直达并高亮命中内容

---

## 5. 同步链路（Google Task）

### 5.1 入口与状态
- 列表页菜单“同步”：`NotesListActivity#onOptionsItemSelected(...)`
- 设置页“立即同步”：`NotesPreferenceActivity#loadSyncButton()`
- 服务入口：`GTaskSyncService`

### 5.2 执行层
- 异步任务：`GTaskASyncTask`
- 核心编排：`GTaskManager#sync(...)`
  - 登录 `GTaskClient#login(...)`
  - 拉取远端列表与任务
  - 双向对账并执行动作

### 5.3 同步决策核心
- 抽象节点：`Node`（Task/TaskList）
- 冲突与增量判定依赖：
  - `local_modified`
  - `sync_id`
  - 远端 `lastModified`
- 动作类型：
  - 本地新增/远端新增
  - 本地更新/远端更新
  - 本地删除/远端删除
  - 冲突更新

### 5.4 元数据机制
- `MetaData` 任务用于保存便签结构化信息（note/data 映射）
- 使远端任务名称与本地完整便签结构可互转

### 5.5 用户可见结果
- 设置页显示同步进度/最后同步时间
- 同步成功后多端便签内容一致
- 同步模式下删除走“回收站”策略，避免直接硬删除

---

## 6. 挂件链路（2x / 4x）

### 6.1 挂件更新入口
- `NoteWidgetProvider_2x` / `NoteWidgetProvider_4x`
- 公共逻辑在 `NoteWidgetProvider#update(...)`

### 6.2 数据读取
- 根据 `widget_id` 查询绑定便签
- 读取：
  - `snippet`（展示文本）
  - `bg_color_id`（挂件背景）

### 6.3 点击行为
- 已绑定便签：点击打开该便签（`ACTION_VIEW`）
- 未绑定便签：点击进入新建（`ACTION_INSERT_OR_EDIT`）

### 6.4 删除挂件
- `NoteWidgetProvider#onDeleted(...)`
- 清理数据库中的 `widget_id` 绑定关系

### 6.5 用户可见结果
- 桌面挂件显示便签摘要与对应颜色
- 编辑便签后挂件内容同步变化

---

## 7. 数据层总览（便于理解上面所有链路）

### 7.1 两张核心表
- `note`：主实体（目录、摘要、颜色、提醒、挂件、同步状态）
- `data`：子数据（正文、清单模式、通话信息等）

### 7.2 Provider 统一入口
- `NotesProvider`
  - `content://micode_notes/note`
  - `content://micode_notes/data`
  - `content://micode_notes/search`

### 7.3 触发器机制
- `NotesDatabaseHelper` 中定义多个 SQLite trigger：
  - 自动维护文件夹 `notes_count`
  - data 改动自动更新 note 摘要 `snippet`
  - 删除 note 自动清理其 data
  - 删除文件夹自动处理子项

### 7.4 用户可见价值
- 列表计数、摘要、层级关系、删除行为能够自动保持一致
- 上层 UI 只需关注业务操作，不必手动维护所有级联字段

---

## 8. 一条完整用户路径示例

“用户新建一条便签 -> 输入内容 -> 设提醒 -> 添加桌面挂件 -> 同步到云端”：

1. 新建：`NotesListActivity -> NoteEditActivity -> WorkingNote.createEmptyNote`
2. 编辑保存：`saveNote -> Note.syncNote`（写入 `note/data`）
3. 设提醒：`setReminder -> AlarmManager.set`
4. 挂件展示：`NoteWidgetProvider.update` 读取摘要并绑定点击意图
5. 云同步：`GTaskSyncService -> GTaskASyncTask -> GTaskManager.sync`
6. 最终用户看到：
   - 列表有该便签
   - 到点有提醒弹窗
   - 桌面挂件展示最新内容
   - 多端同步后内容一致
